Class: Interpreter

Interpreter~Interpreter()

Interpreter class for Tridy commands/statements. A Tridy interpreter includes three parts: a tokenizer, parser, and composer. Each may be stateful to varying degrees. Interacting with this class is all done through the query() method, which automatically pipes the input through these separate components in order.

(async) query(input) → {Array.<Object>}

Interprets a string containing one or more Tridy commands and outputs the results in a JSON object format.
Parameters:
Name Type Description
input String Tridy command(s)/statement(s).
opts.tokenless Boolean Used for internal control flow where it's better to send a pre-processed abstract syntax tree directly as input. Default is false.
opts.accept_carry Boolean True to statefully retain tokens from incomplete statements, false to throw SyntaxError if receiving an incomplete statement. Default is false.
opts.client_mode Boolean True to run as a client, false to run standalone / as a server. Default is false.
opts.host Boolean Server to connect to (only applies if standalone is false). Default is localhost.
opts.port Boolean Port to connect to (only applies if standalone is false). Default is 21780.
opts.timeout Boolean Timeout period (in milliseconds) to wait for responses (only applies if standalone is false). Default is 3000.
opts.tags_key Boolean The key under which tags are imported and exported as. Default is 'tags'.
opts.free_key Boolean The key under which the free data structure is imported and exported as. Default is 'free'.
opts.tree_key Boolean The key under which the free data structure is imported and exported as. Default is 'tree'.
